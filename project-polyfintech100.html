Project Overview

PRUNavigator is a full-stack AI insurance platform built for the PolyFinTech100 API Hackathon. As Team Lead, I guided a 6-member team through Agile sprints — from ideation and architecture to a polished demo — delivering a React.js dashboard that surfaces real-time insurance insights and a personalised coverage scoring engine backed by a secure, relational database.

Key Contributions

Agile Team Leadership

Led a 6-member cross-functional team through structured Agile sprints — managing backlog, sprint reviews, and daily standups to keep delivery on track from concept to working demo.

AI-Driven Dashboard

Architected and built a React.js dashboard that surfaces real-time insurance insights and a personalised coverage score, giving users clear, actionable policy recommendations powered by AI logic.

Secure Backend & Schema Design

Implemented a secure SQLAlchemy schema with strict relational integrity to manage policy and customer data, ensuring data consistency, query efficiency, and protection against injection vulnerabilities.

End-to-End Architecture

Designed the full platform architecture — from REST API contracts between the React frontend and Python backend to the database layer — ensuring each component integrated cleanly under hackathon pressure.

Reflection & Learning

  • Leading a 6-person team under hackathon time constraints reinforced the value of early architectural decisions — a clear API contract between frontend and backend saved significant rework mid-sprint.
  • Building the AI coverage scoring engine showed me how to translate raw policy data into user-facing insights that are both accurate and immediately understandable.
  • Designing the SQLAlchemy schema with strict relational integrity deepened my understanding of data modelling trade-offs between flexibility and consistency in a financial domain.
  • Managing Agile sprints in a real competitive environment taught me to balance technical ambition with delivery discipline — scope decisions matter as much as code quality.
  • With more time, I would expand the AI layer with real-time API feeds from insurers and add a recommendation explainability module so users understand why they receive a particular score.
